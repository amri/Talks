Smaller Handoffs

Problem: By the time we finished delivering a feature, it's already too late for QA to do any acceptance testing (let alone regression testing)

Transfer small tasks frequently. Once a developer finished with one acceptance criteria, hand it over to QA to test just that criteria. The developer could continue with the next acceptance criteria

Logging

Where ? and for what ?

Logs in DB

Location Description
DB://X1PSYSLOG This database log table holds all events (system log) regarding the components (i.e., Successful cold start of process, process stopped, unable to start process)
DB://X1PTRCLOG This database log table holds all events (trace log) regarding the transaction codes (i.e., TC stops at which process)

Logs in /tmp folder

Location Description
/tmp/FEIPI00XX.DMP or /tmp/FEIPO00XX.DMP Look into this log to check if your FEIP line module retrieve all your message
/tmp/FEOPI00XX.DMP or /tmp/FEOPO00XX.DMP Look into this log to check if your FEOP line module retrieve all your message

Logs in /tmp folder (continued)

Location Description
tmp/scmobils.IAMM.IAIT.tmp and /tmp/scmobils.IAMM.ICTX.tmp Look into this log to check if you’ve successfully mapped from MSET to AIT and AIT to L3 component
/tmp/scmobils.IAMM.OCTX.tmp and /tmp/scmobils.IAMM.OAIT.tmp Look into this log to check if you’ve successfully mapped from your L3 component to L3 Comp Context and to AIT

Logs in secondary server

Location Description
/export/home/scmobils/ SCCIMOSSecSvrMobV12/ log/X1IAMM.log Look into this log to check if the process (i.e., IAMM) couldn’t be started or suddenly stop after being brought up
/export/home/scmobils/ SCCIMOSSecSvrMobV12/ log/IAMMlog4j.log Look into this log to see debug/error/warning messages that is being logged by the process or the component (i.e., IAMM).
